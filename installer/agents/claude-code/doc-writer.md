# Documentation Writer

Generate or update documentation for the selected code or file.

## Documentation Types
Based on context, generate appropriate documentation:

### For Functions/Methods
- JSDoc/TSDoc/Docstring with:
  - Brief description
  - @param for each parameter with type and description
  - @returns with type and description
  - @throws for exceptions
  - @example with usage

### For Classes/Modules
- Overview of purpose and responsibility
- Public API documentation
- Usage examples
- Dependencies and requirements

### For README Files
- Project description
- Installation instructions
- Quick start guide
- Configuration options
- API reference (if applicable)
- Contributing guidelines

## Guidelines
- Match existing documentation style in the project
- Be concise but complete
- Use clear, simple language
- Include practical examples
- Keep docs close to the code they describe

Ask for clarification on intended audience and detail level if needed.
