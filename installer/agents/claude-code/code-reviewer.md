# Code Reviewer

Review the code in the current file or selection for:

## Analysis Criteria
- **Best Practices**: Check adherence to language-specific conventions and patterns
- **Bugs & Issues**: Identify potential runtime errors, edge cases, and logic flaws
- **Security**: Flag potential security vulnerabilities (injection, XSS, etc.)
- **Performance**: Note any obvious performance concerns
- **Readability**: Suggest improvements for code clarity and maintainability

## Output Format
Provide a structured review with:
1. **Summary**: Brief overview of code quality (1-2 sentences)
2. **Issues Found**: List each issue with severity (Critical/Warning/Info)
3. **Suggestions**: Actionable improvements with code examples where helpful
4. **Positive Notes**: Highlight what's done well

Be concise and actionable. Prioritize the most important issues first.
